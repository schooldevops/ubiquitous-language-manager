# Task 06: 개발자 연동 가이드 작성

## 목표

개발자가 데이터 사전 기반 검증 도구를 일상 개발 흐름에 적용할 수 있도록 가이드를 작성한다.

## 범위

- OpenAPI 기반 서버 API 사용법
- CLI 사용법
- PR 봇 동작 방식
- CI 실패 대응 방법
- 신규 용어 후보 요청 방법

## 상세 태스크

1. OpenAPI Spec 위치와 사용법을 문서화한다.
2. 생성된 API client 사용법을 문서화한다.
3. 로컬 CLI 검증 명령을 문서화한다.
4. OpenAPI YAML 검증 예시를 작성한다.
5. DDL 검증 예시를 작성한다.
6. DTO 검증 예시를 작성한다.
7. PR 코멘트 예시를 작성한다.
8. CI 실패 시 수정 절차를 작성한다.
9. 신규 용어 후보 등록 절차를 작성한다.
10. Deprecated 용어 대응 절차를 작성한다.

## 산출물

- 개발자 연동 가이드
- CLI 사용 예시
- PR/CI 대응 가이드

## 완료 기준

- 개발자가 표준 위반을 로컬에서 재현하고 수정할 수 있다.
- OpenAPI Spec을 기준으로 서버 API와 클라이언트를 사용할 수 있다.
