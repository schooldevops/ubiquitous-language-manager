# Task 01: RAG 용어 문서 생성

## 목표

용어별 Markdown 문서를 생성하여 RAG 인덱싱의 입력 데이터로 사용한다.

## 범위

- Approved 용어 문서
- Draft/Reviewing 후보 문서
- Deprecated 대체 안내 문서
- 문서 생성 배치

## 상세 태스크

1. 용어 문서 Markdown 템플릿을 정의한다.
2. `TERM_MASTER` 데이터를 문서에 포함한다.
3. `TERM_EXPRESSION` 데이터를 문서에 포함한다.
4. `TERM_ALIAS` 데이터를 문서에 포함한다.
5. `TERM_RELATIONSHIP` 데이터를 문서에 포함한다.
6. 업무 정의와 사용 맥락을 문서 본문에 포함한다.
7. 예시 문장을 문서에 포함한다.
8. Deprecated 용어는 대체 용어를 명시한다.
9. 문서 생성 배치를 구현한다.
10. 문서 생성 결과 검증 테스트를 작성한다.

## 산출물

- 용어별 Markdown 문서
- 문서 생성 배치
- 문서 템플릿

## 완료 기준

- 고객번호 용어 문서가 생성된다.
- 문서에 영문명, DB 컬럼명, API 필드명, 유사어, 금지어, 관련 용어가 포함된다.
