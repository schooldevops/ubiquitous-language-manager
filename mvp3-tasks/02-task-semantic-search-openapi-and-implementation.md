# Task 02: 의미 기반 검색 API 및 구현

## 목표

자연어 질의로 표준 용어 후보를 찾는 의미 기반 검색을 구현한다.

## OpenAPI 필수 조건

서버 구현 전에 `POST /search/semantic` OpenAPI Spec을 먼저 작성한다.

## 상세 태스크

1. `POST /search/semantic` 요청 schema를 정의한다.
2. 질의문, 도메인 필터, 상태 필터, 결과 개수 옵션을 요청에 포함한다.
3. `SemanticSearchResult` 응답 schema를 정의한다.
4. 유사도 점수, 표준 용어, 추천 사유, 차이 설명을 응답에 포함한다.
5. OpenAPI 예시에 `주문이 발생한 날짜` 검색 케이스를 포함한다.
6. 용어 문서 임베딩 생성을 구현한다.
7. 벡터 인덱스 저장 방식을 구현한다.
8. 자연어 질의 임베딩 생성을 구현한다.
9. 유사 용어 후보 검색을 구현한다.
10. Rule Engine 후처리를 적용한다.
11. Deprecated, Draft, Reviewing 상태별 안내 문구를 적용한다.
12. 검색 품질 평가용 테스트 케이스를 작성한다.

## 산출물

- Semantic Search OpenAPI Spec
- 임베딩 생성 로직
- 의미 기반 검색 API
- 검색 품질 테스트 데이터

## 완료 기준

- `주문이 발생한 날짜` 질의에 대해 주문일자와 주문일시를 후보로 제시할 수 있다.
- 후보 간 차이를 설명할 수 있다.
