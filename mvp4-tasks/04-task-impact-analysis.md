# Task 04: 영향도 분석 구현

## 목표

특정 용어 변경 시 영향받는 시스템, DB, API, 코드, 문서, 테스트를 조회한다.

## OpenAPI 필수 조건

영향도 분석 API는 구현 전에 OpenAPI Spec을 먼저 작성해야 한다.

## 상세 태스크

1. `GET /impact/terms/{termId}` OpenAPI Spec을 작성한다.
2. 영향도 응답에 시스템, 테이블, 컬럼, API, DTO, 문서, 테스트를 포함한다.
3. 위험도 계산 결과 schema를 정의한다.
4. 변경 유형 입력 옵션을 정의한다.
5. 고객ID에서 고객번호 변경 예시를 Spec에 포함한다.
6. Graphify 질의를 통해 직접 영향 대상을 조회한다.
7. 2-hop 관계까지 확장 조회하는 옵션을 구현한다.
8. 영향도 위험도를 낮음, 중간, 높음으로 계산한다.
9. 권장 조치 목록을 생성한다.
10. 영향도 분석 화면을 구현한다.
11. 통합 테스트를 작성한다.

## 산출물

- Impact Analysis OpenAPI Spec
- 영향도 분석 API
- 영향도 분석 화면
- 위험도 계산 로직

## 완료 기준

- 고객번호 변경 시 관련 DB 컬럼, API 필드, DTO, 기획서, 테스트 케이스를 조회할 수 있다.
- 변경 위험도와 권장 조치를 함께 제공한다.
