# Task 02: 도메인, 사용자 역할, 권한 정의

## 목표

용어 관리 시스템에서 사용할 도메인, 사용자 역할, 권한 모델을 정의한다.

## 범위

- 업무 도메인 목록 정의
- 사용자 역할 정의
- 역할별 기능 권한 정의
- 승인 프로세스 참여자 정의

## 상세 태스크

1. 초기 도메인 목록을 정의한다.
2. 데이터 스튜어드 역할을 정의한다.
3. 개발 리더 또는 아키텍트 역할을 정의한다.
4. 기획자, 개발자, 운영자, 관리자 역할을 정의한다.
5. 용어 등록 권한을 역할별로 정의한다.
6. 용어 승인 권한을 역할별로 정의한다.
7. 용어 폐기 권한을 역할별로 정의한다.
8. 신규 용어 후보 검토 권한을 정의한다.
9. PR/CI 검증 결과 조회 권한을 정의한다.
10. 권한 검증이 필요한 API 목록을 도출한다.

## 산출물

- 도메인 목록
- 역할 정의표
- 권한 매트릭스
- 승인 프로세스 참여자 목록

## 완료 기준

- 각 기능을 어떤 역할이 사용할 수 있는지 설명할 수 있다.
- 승인, 폐기, 후보 승격의 책임자가 명확하다.
