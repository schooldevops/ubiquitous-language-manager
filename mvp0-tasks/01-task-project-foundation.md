# Task 01: 프로젝트 기반 정리

## 목표

데이터 사전 기반 유비쿼터스 랭기지 시스템의 개발 착수 기준을 확정한다.

## 범위

- 기술 스택 후보 정리
- Backend, Frontend, DB, Search, RAG, Graphify 구성 방식 결정
- 로컬 개발 환경 기준 정의
- 공통 디렉토리 구조 정의
- API-first 개발 원칙 확정

## 상세 태스크

1. Backend 기술 스택을 확정한다.
2. Frontend 기술 스택을 확정한다.
3. 관계형 DBMS를 확정한다.
4. 검색 엔진 또는 벡터 저장소 후보를 정리한다.
5. Graphify 연동 방식을 정의한다.
6. OpenAPI Spec 위치와 파일 명명 규칙을 정한다.
7. 서버 API는 OpenAPI Spec을 먼저 작성한 뒤 구현하는 원칙을 문서화한다.
8. API 요청/응답 공통 포맷을 정의한다.
9. 에러 응답 표준을 정의한다.
10. 로컬 실행, 테스트, 린트 명령을 정의한다.

## 산출물

- 프로젝트 구조 정의서
- 기술 스택 결정 기록
- API-first 개발 원칙
- OpenAPI Spec 관리 규칙
- 로컬 개발 환경 가이드

## 완료 기준

- 개발자가 같은 방식으로 로컬 환경을 구성할 수 있다.
- OpenAPI Spec 파일 위치와 작성 규칙이 확정되어 있다.
- 서버 API 개발 전에 OpenAPI Spec을 작성해야 한다는 기준이 명확하다.
